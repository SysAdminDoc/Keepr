# Google Keep vs Keepr v0.1 - Feature Gap Audit

> Research date: 2026-05-25
> Target: a pixel-close offline clone of Google Keep
> Keepr v0.1 already ships: masonry card grid, pinned/others sections, text notes, checklist notes, 12 Keep colors (light + dark), labels with chip filtering, search across title/body/checklist, archive, trash with restore + empty, light/dark theme, manual ZIP backup/restore.
>
> Status legend: **NOT YET** = unimplemented in v0.1. **PARTIAL** = some shape of it exists but missing key behavior. **DIFFERENT** = exists but diverges from Keep's UX.

---

## 1. Note types beyond text and checklist

| Capability | What it does | Source | Keepr-v0.1 status | Notes / pitfalls |
|---|---|---|---|---|
| **Image notes (multi-image per note)** | One note can hold multiple photos; user can keep adding more via "Add image" in the editor. Photos render in a grid above the title/body. | https://support.google.com/keep/answer/6395566 | **NOT YET** | Pick a single attachment storage model up front (base64 in IndexedDB vs Tauri-side file blobs). Multi-image grid layout differs between 1, 2, 3, 4+ images on Keep web - mimic those breakpoints. |
| **"Grab image text" OCR** | Three-dot menu on an image note runs OCR and appends extracted text into the note body. Works best on printed text; cursive accuracy is poor. Image format restrictions: JPEG/PNG/WEBP/GIF, <10 MB and <25 MP. | https://support.google.com/docs/thread/175253497, https://www.cisdem.com/resource/google-keep-ocr.html, https://www.techrepublic.com/article/how-to-take-advantage-of-optical-character-recognition-in-google-keep/ | **NOT YET** | Offline OCR: ship tesseract.js (English only is acceptable for v1; full lang packs are ~10 MB each). Run in a web worker to avoid blocking the UI. Tauri build can use the Windows OCR API (`Windows.Media.Ocr`) for a smaller binary. |
| **Audio notes (record + transcribe)** | Android-only in the official app. Tap mic in bottom toolbar/widget; speech is transcribed in real time into the note body AND the .m4a recording is attached. Pauses cut off the transcript. Requires online for transcription. iOS reportedly added voice notes despite Google docs claiming Android-only. | https://blogs.lcps.org/inclusivedesign/2022/05/27/audio-recording-and-transcription-in-google-keep/, https://www.computerworld.com/article/1659766/google-keep-android-tips.html, https://www.alphr.com/google-keep-voice-to-text/ | **NOT YET** | Offline-friendly options: (a) record-only (just .webm via MediaRecorder), no transcription; (b) Whisper.cpp WASM for offline ASR (~150 MB model, real wins for a desktop clone). On Android wrapper, use SpeechRecognizer with `EXTRA_PREFER_OFFLINE`. |
| **Drawing / sketch notes** | Standalone drawing note OR a drawing attached to an existing note. Tools: Pen (thin), Marker (thick), Highlighter, Eraser, Selection tool, Grid/rule overlay. **28 colors x 8 sizes**. Canvas auto-extends as you draw to the bottom. Undo/redo. Double-tap eraser to clear all. | https://support.google.com/keep/answer/6395656, https://www.howtogeek.com/how-to-draw-and-sketch-on-notes-using-google-keeps-drawing-tools/, https://www.pcworld.com/article/411373/google-keep-for-chrome-adds-drawing-tools-for-doodling-in-notes.html | **NOT YET** | Web: `<canvas>` with pointer events (pen pressure via PointerEvent.pressure on stylus). Store as SVG paths (compact, infinite resize) AND a rasterized PNG thumbnail for the card grid. Pre-2024 Keep stored drawings as PNG only; in Oct 2024 Google began testing a separate "handwritten notes" type with vector strokes - vector storage is the modern target. |
| **Annotate-on-image** | Pen icon on an embedded image opens the same drawing canvas overlay; strokes saved as part of the note. | https://support.google.com/keep/answer/6395656 | **NOT YET** | Same canvas component as above; just composite on top of image as the background layer. |
| **Handwritten notes (new format, 2024+)** | Separate Material 3 type with Material 3 stroke-width slider and updated color picker; stands alone in the grid rather than being attached to text. | https://9to5google.com/2024/10/23/google-keep-handwritten-notes/ | **NOT YET** | Don't conflate with legacy drawings - keep them as distinct types in the data model from day one so you don't have to migrate later. |
| **Attachment limits per note** | No official cap on image count, but the practical per-note size limit is ~20 MB (text + attachments combined). Files larger than this fail to sync or get truncated. | https://www.quora.com/How-many-notes-can-I-keep-in-Google-Keep, https://support.google.com/docs/thread/163071349 | **NOT YET (no attachments yet)** | Document the chosen cap clearly. IndexedDB on Chromium can hold gigabytes but the UI starts to choke past ~50 thumbnails per card. |

## 2. Reminders / dates

| Capability | What it does | Source | Keepr-v0.1 status | Notes / pitfalls |
|---|---|---|---|---|
| **Time-based reminder** | Bell icon on note. Quick options: "Later today", "Tomorrow", "Next week", "Pick date & time". Fires a notification at the set time AND again 24 h later (Keep auto-renotifies). | https://support.google.com/keep/answer/3187168 | **NOT YET** | Use Web Notifications API + a service worker for browser builds, and tauri-plugin-notification for desktop. Keep the original Keep behavior of "remind again in 24 h if dismissed" - it's a love-or-hate feature; expose a setting. |
| **Recurring reminders** | "Repeat" panel inside Pick date & time supports Daily/Weekly/Monthly/Yearly/Custom (interval + weekday picker + end date). | https://support.google.com/keep/answer/3187168 | **NOT YET** | Store as RRULE strings (RFC 5545) so import/export to Calendar/ICS later is trivial. |
| **Location-based reminders** | **DISCONTINUED in late 2025.** Keep no longer creates or fires "when I arrive at Home" reminders. Existing ones got migrated to Tasks (where they also no longer fire on location). | https://en.wikipedia.org/wiki/Google_Keep, https://support.google.com/assistant/thread/289543 | **N/A - skip** | An offline desktop clone doesn't need this. If a mobile wrapper is built, geofencing via Android `LocationManager` is doable but extremely battery-hungry. |
| **Default reminder times (Morning/Afternoon/Evening)** | Settings page lets the user define what "morning", "afternoon", "evening" mean (default ~8 AM / 1 PM / 6 PM). Quick-add buttons in the reminder picker reference these. | https://support.google.com/keep/answer/3187168, https://support.google.com/keep/answer/6358550 | **NOT YET** | Trivial - three time pickers in Settings. Default to 8/13/18 like Keep does. |
| **Reminders sidebar section** | Left rail has a "Reminders" entry that shows all notes with unfired reminders, sorted by reminder time. Overdue reminders sit at the top with a red-tinted label like "Yesterday 3:00 PM". | personal experience (Keep web), https://support.google.com/keep/answer/3187168 | **NOT YET** | Easy to add - it's just a filter view. The overdue styling is the part to get right. |
| **Migration to Google Tasks (2025+)** | Google is actively moving Keep reminders into Tasks; reminder UI in Keep still works but the data lives in Tasks now. Wear OS lost the ability to set reminders at all. | https://support.google.com/tasks/answer/16540694, https://www.phonearena.com/news/the-latest-google-keep-update-is-breaking-a-key-feature-for-wear-os-users_id177636 | **N/A - offline clone** | Note for context only - means a clone should keep reminders inside the note model, not split them out, to stay simpler than the real product. |
| **Snooze on the notification** | When the reminder fires on Android, the notification has Snooze/Mark done actions. Snooze offers 10 min / 1 hour / Tomorrow morning / Pick. | personal experience (Keep Android), https://9to5google.com/2020/07/27/set-reminder-notification-android/ | **NOT YET** | Web Notifications API does not support custom action buttons across all browsers - check before promising. Tauri can use native notification actions on Windows 10/11 via `tauri-plugin-notification`. |

## 3. Collaboration

| Capability | What it does | Source | Keepr-v0.1 status | Notes / pitfalls |

**Implementation pitfalls:**
- `j` `k` `n` `p` `x` `e` `f` `c` `l` fire without modifiers - must be suppressed when focus is in any text input. Use a `useGlobalHotkey` hook that ignores events whose target is `INPUT`, `TEXTAREA`, or `[contenteditable]`.
- `#` is a Shift+3 chord - watch for non-US keyboard layouts (e.g. UK has `#` directly on a key).
- Provide a help overlay (the `?` shortcut) that lists everything. Keep's own overlay is the gold standard.

## Summary of gaps vs Keepr v0.1

**Highest-impact missing surface area, ranked by user pain on day 1:**

1. **Image notes + drawings** (section 1) - Keep is fundamentally multimodal. Text-only is "kinda like Keep" at best.
2. **Reminders with notifications** (section 2) - Keep's reminder bell is its second most-used button after the checkbox. Without it Keepr is just a sticky-notes app.
3. **Multi-select + bulk actions** (section 6) - Once a user has 50+ notes, organization without bulk operations is unbearable.
4. **Search filter chips** (section 7) - Already partway there; adding type/color/label chips on top of existing text search is small effort, big perceived polish.
5. **Keyboard shortcuts overlay** (section 9) - Free power-user appeal; ~1 day of work to cover the canonical list.
6. **Drag-to-reorder + Custom sort** (section 6) - Recent Keep addition; users have been asking for it for a decade.
7. **Background image patterns** (section 5) - Keepr can match the 9 textured backgrounds AND make them sync everywhere (Keep doesn't sync them to web).
8. **Takeout-compatible import/export** (section 15) - Lets real Keep users move to Keepr in one drag-and-drop.
9. **Make a copy / Show-Hide checkboxes / Move checked to bottom / Indent sub-items** (sections 4, 5) - All small features that together close the editor parity gap.
10. **Edit/created timestamps in editor footer** (section 5) - Trivially small, expected by every user.

**Win opportunities (things Keep does poorly and Keepr can do better):**

- Show note counts next to labels in the sidebar (section 11).
- Pin labels to the top of the sidebar (section 11).
- Persistent undo history (survives closing a note) (section 5).
- Rich version history with diff and one-click rollback (section 5).
- Background images sync everywhere (section 5).
- Configurable trash retention (not hardcoded 7 days) (section 13).
- "Days remaining" badge on trashed notes (section 13).
- High-contrast theme variant (section 16).
- A-Z sort option in addition to date sorts (section 6).
- Inline `#labelname` auto-tag with strong autocomplete (section 11).

**Things deliberately skip for an offline clone:**

- Real-time co-edit / collaborators (section 3).
- Location-based reminders (deprecated even by Google) (section 2).
- Google Docs sidebar integration (section 17).
- Wear OS app (section 17).
- Family group sharing (section 3).
- "Things" auto-categorization in search (requires an ML model) (section 7).
